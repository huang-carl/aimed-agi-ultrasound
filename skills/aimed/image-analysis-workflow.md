# Skill: image-analysis-workflow

## 描述
超声图像分析标准工作流，从图像上传到诊断结论的完整流程。

## 触发条件
- 用户上传超声图像
- 调用图像分析相关 API
- 需要执行图像分割/特征提取时

## 标准工作流

### Phase 1: 图像预处理
```
1. 图像接收
   - 支持格式：DICOM, PNG, JPG, TIFF
   - 自动检查图像质量（清晰度、完整性）
   - 记录图像元数据（设备、参数、时间）

2. 图像标准化
   - 统一分辨率（建议 512×512 或 1024×1024）
   - 对比度增强（CLAHE 算法）
   - 噪声过滤（高斯滤波/中值滤波）
   - 感兴趣区域（ROI）自动标注

3. 质量检查
   - 图像清晰度评分（≥0.7 为合格）
   - 关键结构可见性检查（胃壁分层/胰腺轮廓）
   - 不合格图像提示重新拍摄
```

### Phase 2: 特征提取
```
1. 灰阶特征
   - 胃壁/胰腺形态测量（厚度、面积、体积）
   - 回声特征分析（低/等/高回声比例）
   - 层次结构完整性（胃壁 5 层是否清晰）

2. 造影特征
   - 时间 - 强度曲线（TIC）分析
   - 增强模式分类（早/同步/延迟）
   - 增强强度量化（高/等/低）
   - 廓清模式识别（早/晚/无/向心）

3. 特殊征象
   - 伪足征检测
   - 辐射状血管识别
   - 不规则边界检测
```

### Phase 3: AI 分析
```
1. 模型推理
   - 输入：预处理后的图像 + 提取的特征向量
   - 模型：Hermes V2.1 诊断模型
   - 输出：诊断分型（I-V 型）+ 恶性概率

2. 置信度评估
   - 模型置信度评分（≥0.85 为高置信度）
   - 特征重要性排序（Top 3 决定性特征）
   - 不确定性量化（置信区间）

3. 异常检测
   - 图像与报告一致性检查
   - 特征冲突检测（如：早增强但无廓清）
   - 低置信度预警（<0.7 提示人工复核）
```

### Phase 4: 报告生成
```
1. 结构化输出
   - 遵循 diagnostic-report-standard 模板
   - 关键数据高亮显示
   - 图像标注叠加（可选）

2. 多格式导出
   - Markdown（默认）
   - PDF（带印章/签名位）
   - Word（可编辑）

3. 存档与追溯
   - 报告 ID 生成（UUID）
   - 模型版本记录
   - 分析时间戳
```

## API 调用示例

```python
# 图像上传 + 分析
POST /api/v1/diagnosis
{
    "image": "[base64 或文件路径]",
    "modality": "CEUS",  # 超声造影
    "region": "stomach",  # 胃部/胰腺
    "patient_id": "anon_001"
}

# 响应
{
    "report_id": "rpt_20260504_001",
    "diagnosis_type": "IV",
    "malignancy_probability": 0.78,
    "key_features": ["早增强", "早廓清", "伪足征"],
    "confidence": 0.82,
    "recommendation": "进一步检查"
}
```

## 注意事项
1. **图像质量**：低质量图像会导致分析失败，需提前检查
2. **模型版本**：每次分析记录使用的模型版本，便于追溯
3. **人工复核**：低置信度结果（<0.7）必须提示人工复核
4. **数据脱敏**：图像元数据中的患者信息需脱敏处理
5. **性能监控**：记录分析耗时，优化慢查询
