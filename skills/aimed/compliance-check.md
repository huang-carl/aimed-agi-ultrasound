# Skill: compliance-check

## 描述
合规边界自动检查，确保 AIMED 系统在 Phase 1 阶段严格遵守科研工具定位，不触碰临床诊断红线。

## 触发条件
- 生成诊断报告前
- 对外发布内容前
- 用户请求超出权限范围的服务时
- 定期检查系统合规状态

## Phase 1 合规边界（2026 Q2-Q4）

### ⚠️ 严禁行为（红线）
```
1. 对外宣称"诊断"
   - ❌ 禁止使用"AI 诊断系统"、"自动诊断"等表述
   - ✅ 应使用"AI 辅助分析"、"科研工具"、"技术支持"

2. 直接面向患者
   - ❌ 禁止患者直接上传图像获取报告
   - ✅ 必须通过医疗机构/科研合作单位中转

3. 未经伦理审批
   - ❌ 禁止使用未经伦理委员会审批的真实患者数据
   - ✅ 仅使用脱敏 Mock 数据或已获审批的真实数据

4. 商业化收费
   - ❌ 禁止以诊断服务名义向患者/医院收费
   - ✅ 可收取技术支持费、科研合作费（需合同）
```

### ✅ 允许行为（绿灯）
```
1. 科研合作
   - 与医院/高校签订科研合作协议
   - 提供技术支持，不参与临床决策

2. 技术验证
   - 内部测试 AI 模型准确率
   - 对比 AI 分析与医师诊断的一致性

3. 学术交流
   - 发表学术论文（需脱敏）
   - 参加学术会议展示技术

4. 合规数据采集
   - 通过合作单位合规采集标注数据
   - 数据脱敏后用于模型训练
```

## 自动检查清单

### 报告合规检查
```python
def check_report_compliance(report_text):
    """检查报告是否包含合规声明"""
    checks = [
        ("免责声明", "不能替代专业医师诊断" in report_text),
        ("辅助定位", "AI 辅助" in report_text or "技术支持" in report_text),
        ("非诊断表述", "诊断" not in report_text.replace("辅助诊断", "")),
        ("版本记录", "模型版本" in report_text),
    ]
    return all([check[1] for check in checks])
```

### 对外内容检查
```python
def check_public_content(content):
    """检查对外发布内容是否合规"""
    forbidden_words = [
        "诊断系统", "自动诊断", "替代医生", "临床诊断",
        "患者直接", "收费诊断"
    ]
    required_phrases = [
        "AI 辅助", "科研工具", "技术支持", "不能替代医师"
    ]
    
    violations = [word for word in forbidden_words if word in content]
    missing = [phrase for phrase in required_phrases if phrase not in content]
    
    return {
        "compliant": len(violations) == 0 and len(missing) == 0,
        "violations": violations,
        "missing": missing
    }
```

## 合规声明模板

### 网站/宣传材料
```
【声明】
AIMED 充盈视界平台目前处于科研验证阶段，
所提供的 AI 辅助分析功能仅供医疗专业人员参考，
不能替代专业医师的临床诊断。

最终诊断请以临床医师综合判断为准。
```

### 诊断报告
```
【免责声明】
本报告由 AI 辅助生成，仅供参考，
不能替代专业医师诊断。
最终诊断请以临床医师综合判断为准。
```

## 定期检查机制
```
频率：每周
检查项：
1. 对外宣传材料是否包含禁用词汇
2. 诊断报告是否包含完整免责声明
3. 数据采集流程是否符合伦理要求
4. 用户协议是否明确科研工具定位
5. 合作单位资质是否有效
```

## 违规处理流程
```
1. 发现违规 → 立即暂停相关功能
2. 评估影响 → 确定影响范围和严重程度
3. 整改 → 修改内容/流程
4. 复核 → 合规检查通过后方可恢复
5. 记录 → 记录违规事件和整改措施
```

## 注意事项
1. **动态更新**：合规边界随 Phase 推进动态调整（Phase 2/3 会有变化）
2. **法律咨询**：重大决策前咨询医疗法律专家
3. **培训机制**：定期对团队成员进行合规培训
4. **文档留存**：所有合规检查记录存档备查
