## 🎯 AIMED 项目需求分析

### 核心任务
1. **超声影像描述解析** - 理解影像特征、病灶描述
2. **诊断推理** - 基于影像 + 病历给出诊断意见
3. **报告生成** - 结构化医疗报告输出
4. **鉴别诊断** - 多可能性分析与置信度评估
5. **中文医疗文本** - 病历、报告、医嘱等

### 关键要求
| 维度 | 重要性 | 说明 |
|------|--------|------|
| **中文理解** | ⭐⭐⭐⭐⭐ | 医疗数据 100% 中文 |
| **医学知识** | ⭐⭐⭐⭐⭐ | 超声诊断专业性极强 |
| **推理能力** | ⭐⭐⭐⭐ | 鉴别诊断需要逻辑推理 |
| **上下文窗口** | ⭐⭐⭐⭐ | 需处理完整病历 + 影像描述 |
| **响应速度** | ⭐⭐⭐ | 临床场景需要快速出报告 |
| **成本控制** | ⭐⭐⭐ | 免费期结束后需考虑成本 |

## 🏆 最佳模型推荐

### 方案 A：阿里云为主 + NVIDIA 为辅（推荐）

| 任务类型 | 推荐模型 | 理由 |
|----------|----------|------|
| **日常诊断推理** | `qwen-max` | 中文医疗理解最佳，推理能力强 |
| **快速筛查** | `qwen-plus` | 速度均衡，成本低 |
| **长病历分析** | `qwen-long` | 10M 上下文，可处理完整病历 |
| **复杂鉴别诊断** | `meta/llama-3.3-70b-instruct` | 70B 参数，推理深度好 |
| **快速原型验证** | `qwen-flash` | 开发调试阶段快速迭代 |

**路由策略：**
```python
# 智能路由逻辑
if task == "快速筛查":
    model = "qwen-flash"  # 速度优先
elif task == "日常诊断":
    model = "qwen-plus"   # 均衡选择
elif task == "复杂诊断":
    model = "qwen-max"    # 效果优先
elif task == "长病历分析":
    model = "qwen-long"   # 上下文优先
elif task == "鉴别诊断":
    model = "nvidia/llama-3.3-70b"  # 英文模型推理优势
```

### 方案 B：纯阿里云（简化架构）

**优点：** 架构简单、中文优化好、本地化支持
**缺点：** 缺乏多样性验证、单点依赖
