## 📊 分级体系

### 🔓 公开级 (Public)
*任何人可访问，对外展示用*

| 类别 | 内容 | 示例 |
|------|------|------|
| **官网内容** | 产品介绍/解决方案/技术说明 | `static/index.html`, `static/products.html` |
| **演示 Demo** | 诊断演示页面 | `static/demo.html` |
| **项目说明** | README/架构概览/快速入门 | `README.md`, `docs/medical/aimed-core-logic.md` |
| **技术文档** | API 文档/开发指南 | `docs/development/API_EXAMPLES.md` |
| **部署文档** | 部署指南/配置说明 | `docs/deployment/` |
| **V1.0 定义** | 系统/算法/前端定义 | `docs/medical/SYSTEM_V1.0.md` |

**访问权限：** 任何人（公开）  
**GitHub：** 可提交  
**网站：** 公开访问  
**服务器：** 公开目录

---

### 🔑 授权级 (Authorized)
*需要授权才能访问，合作伙伴/投资人/临床医生*

| 类别 | 内容 | 示例 |
|------|------|------|
| **战略规划** | 路线图/商业模式/融资材料 | `docs/strategy/答辩核心要点.md` |
| **临床数据** | 脱敏病例/训练数据 | `data/cases/`（脱敏后） |
| **合作协议** | 科研合作/医院协议 | `attachments/`（合作协议类） |
| **AI 平台** | 医生工作台/患者咨询台 | `static/portal/`（需登录） |
| **内部报告** | 部署报告/验证报告 | `docs/deployment/部署完成报告.md` |

**访问权限：** 授权用户（登录/密钥）  
**GitHub：** 私有仓库或加密提交  
**网站：** 需登录  
**服务器：** 受保护目录

---

### 🔒 保密级 (Confidential)
*仅供内部查阅，核心机密*

| 类别 | 内容 | 示例 |
|------|------|------|
| **API 密钥** | 模型 API Key/数据库密码 | `config.py`, `config/*.json` |
| **智能体配置** | 小超/小菲配置文件 | `config/xiaochao-config.json` |
| **内部记忆** | MEMORY.md/SOUL.md/USER.md | 工作区核心文件 |
| **原始数据** | 未脱敏病例/患者信息 | `data/cases.db` |
| **SSL 证书** | 私钥/证书文件 | `/etc/letsencrypt/` |
| **服务器配置** | Nginx 配置/系统服务 | `/etc/nginx/`, `deploy/hermes.service` |
| **训练模型** | 模型权重/向量索引 | `data/vectors/` |
| **个人文件** | skytop-dept/ 个人工作区 | 非项目文件 |

**访问权限：** 仅授权开发人员  
**GitHub：** **禁止提交**（.gitignore 过滤）  
**网站：** 不公开  
**服务器：** 权限控制（chmod 600）

## 🔐 敏感信息处理

### API 密钥管理
```
✅ 正确做法：
- 使用环境变量
- 存储在 config.py（不提交）
- 使用密钥管理服务（阿里云 KMS）

❌ 错误做法：
- 硬编码在代码中
- 提交到 GitHub
- 存储在公开文档中
```

### 病例数据管理
```
✅ 正确做法：
- 脱敏处理（移除患者姓名/身份证）
- 本地存储（不提交 GitHub）
- 加密传输（HTTPS）
- 访问日志记录

❌ 错误做法：
- 明文存储患者信息
- 提交到公开仓库
- 未加密传输
```
