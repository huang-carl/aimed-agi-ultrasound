## 🚀 **5 分钟快速开始**

### 1️⃣ 系统已就绪

```bash
# 检查服务状态
curl http://127.0.0.1:18790/health

# 查看 Swagger 文档
浏览器打开：http://127.0.0.1:18790/docs
```

### 2️⃣ 上传第一个病例

**方式一：通过 API**
```bash
curl -X POST http://127.0.0.1:18790/api/v1/cases/upload \
  -F "organ=胃" \
  -F "patient_id=P001" \
  -F "image=@stomach.jpg" \
  -F "image_description=胃窦部黏膜充血水肿，可见点状糜烂"
```

**方式二：通过钉钉**
- 直接发消息给小超同学
- 格式：`诊断：胃窦部黏膜充血水肿`

### 3️⃣ 查看统计

```bash
curl http://127.0.0.1:18790/api/v1/cases/stats/statistics
```

## 📋 **核心功能**

### 1. AI 诊断

**支持器官：** 胃、胰腺

**使用示例：**
```
"帮我诊断：胃窦部黏膜充血水肿，可见点状糜烂"
```

**输出：**
- 诊断结论
- 置信度 (0-1)
- 鉴别诊断
- 检查建议
- 治疗建议

### 2. 病例管理

**功能：**
- 📤 图像上传
- 🗂️ 自动分类（器官/日期）
- 💾 本地存储（私有）
- 📊 统计报表
- 📥 导出 JSON

**存储路径：**
```
/root/.openclaw/workspace/data/cases/
├── stomach/      # 胃部
├── pancreas/     # 胰腺
└── cases.db      # 数据库
```

### 3. 双渠道支持

| 渠道 | 智能体 | 状态 |
|------|--------|------|
| 钉钉 | 小超同学 | ✅ 已上线 |
| 飞书 | 小菲同学 | ⏳ 配置中 |

## 🔧 **常用 API**

### 诊断接口
```bash
POST /api/v1/diagnosis/diagnose
{
  "organ": "胃",
  "image_description": "胃窦部黏膜充血水肿",
  "model_preference": "nvidia"
}
```

### 上传病例
```bash
POST /api/v1/cases/upload
FormData: organ, image, image_description
```

### 获取统计
```bash
GET /api/v1/cases/stats/statistics
```

### 导出报告
```bash
GET /api/v1/cases/export/report
```

## 🛠️ **环境配置**

### 环境变量 (.env)
```bash
# NVIDIA API
NVIDIA_API_KEY=nvapi-xxx
NVIDIA_MODEL=meta/llama-3.3-70b-instruct

# 服务配置
PORT=18790
MOCK_MODE=false
MODEL_ROUTING=nvidia

# 存储配置（可选）
ALIYUN_OSS_ENABLED=false  # 建议关闭
```

### 启动服务
```bash
cd /root/.openclaw/workspace
python3 main.py
```
