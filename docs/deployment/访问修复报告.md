## 🔧 修复内容

### 问题描述
Nginx 配置中 `/` 和 `/static/` 路径冲突，导致静态文件路由不清晰。

### 修复方案
优化 Nginx 配置，简化路由规则：
- **根路径 `/`** → 直接返回 `index.html`
- **HTML 页面** → `/xxx.html` 直接访问
- **静态资源** → 图片/CSS/JS 等自动缓存

### 配置变更
**文件：** `/etc/nginx/conf.d/aius.conf`

**主要改动：**
1. 移除 `/static/` 路径别名
2. 根位置直接指向 `static/` 目录
3. 简化 HTML 页面访问路径
4. 添加 404 错误页面支持

## 📋 Nginx 配置优化

### 路由规则
```nginx
# 根路径 - 直接返回 index.html
location = / {
    root /root/.openclaw/workspace/static;
    try_files /index.html =404;
}

# HTML 页面
location / {
    root /root/.openclaw/workspace/static;
    try_files $uri $uri/index.html $uri.html =404;
}

# API 代理
location /api/ {
    proxy_pass http://127.0.0.1:18790;
    # ... 代理配置
}

# 静态资源缓存
location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2)$ {
    root /root/.openclaw/workspace/static;
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

### 安全配置
- ✅ HSTS (Strict-Transport-Security)
- ✅ X-Frame-Options (防点击劫持)
- ✅ X-Content-Type-Options (防 MIME 嗅探)
- ✅ HTTP 自动重定向到 HTTPS
