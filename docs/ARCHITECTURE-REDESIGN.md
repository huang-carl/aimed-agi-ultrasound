## 一、当前问题诊断

### 现状：12 个 portal 页面 + 10 个官网页面，定位模糊

| 问题 | 说明 |
|------|------|
| 🔀 定位模糊 | portal 既是入口、又是工作台、又是管理台，角色交叉 |
| 🔄 功能重叠 | 机构管理台 vs 协作研究台（都有项目/成员）；demo.html vs diagnosis-demo.html |
| 📦 归属不清 | 区块链身份（基础设施 vs 功能页面）、知识库（公共服务 vs 平台功能） |
| 🗑️ 冗余文件 | products/solutions/technology 与首页重叠；longnao 3 个模板与 AIMED 无关 |
| 📂 目录混乱 | portal/ 里混着登录、工作台、管理台、公共服务 |

### 核心矛盾
**一个 portal 目录试图承载三种角色（外部用户 + 机构成员 + 内部管理员），导致页面互相干扰。**

## 五、关键设计决策

### 5.1 为什么分两个登录？

| 维度 | 服务平台登录 | 管理平台登录 |
|------|-------------|-------------|
| URL | `/service/login.html` | `/admin/login.html` |
| 身份类型 | 个人身份（医生/患者/研究者）+ 机构身份 | 管理员身份 |
| 权限范围 | 诊断、协作、学术、资产 | 运营、分配、监控、配置 |
| 视觉风格 | 蓝色系（服务/温暖） | 深色系（管理/专业） |

**共享：** DID 身份系统、JWT 认证机制

### 5.2 身份体系（保持不变）

```
个人 DID（基础）
├── 医生 → 诊断工作台 + 协作研究 + 学术
├── 患者 → 患者咨询
├── 研究者 → 协作研究 + 模型训练
└── 机构成员 → 加入机构 + 收益分配

机构身份（扩展）
├── 医院 / 体检中心 / 科研机构
├── 机构管理员 → 管理平台（机构维度）
└── 成员管理 + 分配规则
```

### 5.3 数字资产归属

- **服务平台 `/service/assets.html`** → 个人视角（我的钱包、我的收益、我的 NFT）
- **管理平台 `/admin/distribution.html`** → 管理视角（分配规则、结算对账、智能合约）

### 5.4 知识库归属

- **公开版 `/docs/knowledge.html`** → 50+ 文章 + FAQ（无需登录）
- **服务版 `/service/` 内嵌** → 登录后的高级内容（研究报告、论文全文）

## 八、对比总结

| 维度 | 当前 | 重新设计后 |
|------|------|-----------|
| 入口 | 1 个 portal（模糊） | 2 个平台（清晰） |
| 页面数 | 22 个（含冗余） | 18 个（精简） |
| 定位 | 混合（服务+管理+入口） | 分离（服务平台 vs 管理平台） |
| 导航 | 交叉（角色卡片 vs 功能页面） | 分层（首页 → 功能页） |
| 扩展性 | 低（加功能不知放哪） | 高（按平台归类） |
| 内部管理 | 无（角色模糊） | 5 家公司独立管理台 |
