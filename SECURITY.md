# 安全策略 (Security Policy)

## 📋 安全承诺

AIMED Agent Swarm 项目重视安全性，我们致力于为用户提供安全可靠的软件。

## 🚨 报告安全漏洞

### 如何报告

发现安全漏洞？请通过以下方式报告：

**📧 邮箱：** aimed@aius.xin

**邮件标题：** `[Security] <漏洞简述>`

**邮件内容应包含：**
- 漏洞类型描述
- 复现步骤
- 影响范围
- 修复建议（如有）

### 响应时间

| 严重程度 | 响应时间 | 修复时间 |
|---------|---------|---------|
| 🔴 严重 | 24 小时内 | 7 天内 |
| 🟠 高危 | 48 小时内 | 14 天内 |
| 🟡 中危 | 7 天内 | 30 天内 |
| 🟢 低危 | 14 天内 | 下个版本 |

### 漏洞分级

| 级别 | 标准 |
|------|------|
| **严重** | 远程代码执行、数据泄露、认证绕过 |
| **高危** | 权限提升、SQL 注入、XSS |
| **中危** | 信息泄露、CSRF、不安全的配置 |
| **低危** | 轻微的信息泄露、最佳实践偏离 |

## 🛡️ 安全最佳实践

### 部署环境

```bash
# ✅ 生产环境必须修改的配置
DEBUG=false
ALLOWED_ORIGINS=https://your-domain.com
JWT_SECRET_KEY=<强随机密钥>
LOG_LEVEL=WARNING

# ❌ 禁止使用默认配置
# JWT_SECRET_KEY=your_jwt_secret_key_here_change_in_production
```

### API Key 管理

- ✅ 使用环境变量存储 API Key
- ✅ 定期轮换密钥（建议 90 天）
- ✅ 不同环境使用不同的 Key
- ❌ 不要将 Key 提交到 Git
- ❌ 不要在日志中打印 Key

### 数据安全

- ✅ 患者数据脱敏处理
- ✅ 医疗影像加密存储
- ✅ 访问日志记录
- ❌ 不存储原始患者身份信息

## 🔍 安全审计

### 依赖扫描

使用以下工具定期扫描依赖漏洞：

```bash
# pip-audit
pip install pip-audit
pip-audit -r requirements.txt

# safety
pip install safety
safety check -r requirements.txt
```

### 代码审计

- 定期审查敏感代码（认证、授权、数据处理）
- 使用静态分析工具（flake8, pylint, bandit）

```bash
# 安装安全扫描工具
pip install bandit

# 运行安全扫描
bandit -r agents/ routers/ -f json -o bandit-report.json
```
